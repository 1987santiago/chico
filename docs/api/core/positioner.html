Cómo se usa

El Positioner es un componente que tiene como función posicionar elementos con respecto a un contexto.

El contexto puede ser cualquier elemento del DOM o el area visible de la ventana del navegador (o viewport).

Existen dos formas de iniciar el componente:

1. Como objeto global:

ui.positioner();

2. Desde una instancia de componente:

component.position();

Configuración

La mayoría de los componentes pueden recibir parámetros de configuración, que modifican el comportamiento por defecto.

Objeto de configuración:

{
	element: $(element),		//  Element to be positioned
	context: $(element),		//  Context from which the element is positioned. Default: viewport
	offset: string,			 //  A string with X and Y coordinates (in pixels) separated by space (i.e.: "10 0"). Default "0 0"
	points: string,			 //  A string with element and context shared axis, separated by space (i.e.: "lt lb"). Default "cm cm" (centered)
	hold: boolean			   //  Hold element's position. Default "false"
}

Ejemplo como objeto global:

ui.positioner({
	element: $(".foo"),
	context: $(".context"),
	offset: "10 0",
	points: "cm cm"
});

Ejemplo de configuración desde una instancia de componente:

component.position({
	context: $(".context"),
	offset: "10 0",
	points: "cm cm"
});

Points

Como parte de la configuración, el Positioner puede recibir una referencia a los puntos en común por los cuales el contexto y el elemento deben posicionarse.

Para especificar este comportamiento se definieron una serie de puntos con la siguiente estructura:

points: "lt lb"

Las primeras dos letras del string representan el punto de referencia del elemento a posicionar, mientras que las segundas dos letras representan el punto de referencia del contexto con respecto al cual se posicionará el elemento. Ambos grupos están separados por un espacio.

La primera letra de cada una de estas dos agrupaciones, representa el eje horizontal (left, center o right), y la segunda letra representa el eje vertical (top, middle o bottom).

Las posibles combinaciones de posicionamiento son las siguientes:

Puntos Descripción Ubicación del elemento con respecto al contexto
cm cm Element center-middle point = Context center-middle point Center
lt lb Element left-top point = Context left-bottom point Bottom
lb lt Element left-bottom point = Context left-top point Top
rt rb Element right-top point = Context right-bottom point Bottom
rb rt Element right-bottom point = Context right-top point Top
lt rt Element left-top point = Context right-top point Right
lm lm Element left-middle point = Context left-middle point Right
lt lt Element left-top point = Context left-top point Right

Para tener en cuenta: En próximas versiones de Chico, se agregarán más combinaciones de posicionamiento.

Offset

Dentro de las posibles configuraciones del Positioner, existe la posibilidad de desplazar un elemento de su posición original, separandolo de los puntos que comparte con el contexto (Ver ejemplo 10).

Para especificar este comportamiento se utiliza un string con las coordenadas X e Y (en pixeles) separadas por un espacio:

offset: "10 0"

Inteligencia de posicionamiento

Cuando se detecta que el elemento posicionado no tiene espacio para mostrarse dentro de la ventana del navegador, el Positioner automáticamente modifica la orientación del elemento.

Este comportamiento puede verse en el ejemplo 1, haciendo scroll hasta que el elemento no tenga espacio para mostrarse debajo de su contexto. De esa forma, se desplaza el elemento hacia arriba del contexto.

Es así entonces como un elemento siempre se va a mantener tan visible como sea posible, excepto si se le especifica el comportamiento contrario (Ver ejemplo 2).

Para evitar que el elemento se reposicione cuando no tiene espacio dentro del area visible, se utiliza:

hold: true
